     Plan: Python-configurable initial pose for experiments 1–11

     Context

     Experiments 1–11 each hardcode the robot's starting position and
     orientation in CreateAndAddEpucks via CreateEpuck(label, x, y, theta).
     Users can't change this without editing C++. Since we now have Python
     controller files for each experiment, the cleanest solution is to let users
     declare INIT_X, INIT_Y, INIT_THETA at the top of each Python file, and
     have C++ read those values before placing the robot.

     Approach

     1. Add ReadInitPose static helper to CPythonControllerBase

     controllers/pythoncontrollerbase.h — add to public section:
     static void ReadInitPose(const char* pch_script,
                              double& x, double& y, double& theta);

     controllers/pythoncontrollerbase.cpp — implement at the bottom:
     void CPythonControllerBase::ReadInitPose(const char* pch_script,
                                               double& x, double& y, double& 
     theta)
     {
         x = y = theta = 0.0;
         char cmd[512];
         snprintf(cmd, sizeof(cmd),
             "python3 -c \"g={}; exec(open('%s').read(),g); "
             "print(g.get('INIT_X',0.0), g.get('INIT_Y',0.0), 
     g.get('INIT_THETA',0.0))\"",
             pch_script);
         FILE* f = popen(cmd, "r");
         if (!f) return;
         fscanf(f, "%lf %lf %lf", &x, &y, &theta);
         pclose(f);
     }

     2. Update CreateAndAddEpucks in all 11 experiment files

     Pattern (replace hardcoded coords with ReadInitPose):
     void CTestWheelsExp::CreateAndAddEpucks(CSimulator* pc_simulator)
     {
         double px, py, ptheta;
         CPythonControllerBase::ReadInitPose("python/wheels_controller.py", px,
     py, ptheta);
         char label[100];
         for (int i = 0; i < m_nRobotsNumber; i++) {
             sprintf(label, "epuck%.4d", i);
             CEpuck* pcEpuck = CreateEpuck(label, px, py, ptheta);
             pc_simulator->AddEpuck(pcEpuck);
         }
     }
